## 2. Reproducibility Policy

### 2.1 Bitwise Reproducibility
All numerical results are **bitwise reproducible** when executed within the provided Conda environment with the following constraints:
- **Single-threaded BLAS:** `OPENBLAS_NUM_THREADS=1` (enforced automatically)
- **Deterministic hashing:** `PYTHONHASHSEED=0`
- **Fixed random seeds:** All stochastic solvers use `np.random.seed(42)`

### 2.2 Temporal Dependency Decay
To mitigate software rot, this artifact:
- Uses **locked dependencies** via `environment.yml` (Conda) and `requirements.lock.txt` (pip)
- Avoids deprecated APIs (e.g., `pkg_resources` → `importlib.metadata`)
- Provides **Docker fallback** (`Dockerfile`) for long-term archival

### 2.3 Epistemic Traceability
All solver executions generate **epistemic trace logs** (`experiments/data/epistemic/*.json`) recording:
- Spectral gap ($\lambda_2$)
- Inverse participation ratio (IPR)
- Convergence diagnostics
- Thermodynamic consistency checks

## 8. Known Limitations

### 8.1 Mesh-Induced Spectral Localization
The GGNI spectral heuristic exhibits degraded performance on highly heterogeneous unstructured meshes (e.g., `Geometria_Transfinita`) due to **eigenvector localization**. This is a well-documented phenomenon in spectral graph theory and constitutes the primary direction for future work (multilevel graph coarsening).

### 8.2 Simulated Annealing Runtime
The SA baseline requires **orders of magnitude more computation** (minutes vs. milliseconds for GGNI) due to its iterative Monte Carlo nature. This is expected and actually strengthens the paper's narrative about GGNI's efficiency.

### 8.3 Elastic Relaxation in SA
Simulated Annealing achieves energies **below the Max-Flow combinatorial bound** by allowing partial bond breaking and elastic strain redistribution. This is physically admissible but not captured by the rigid-block kinematics assumed in the Max-Flow formulation.

## 9. Troubleshooting

### 9.1 Common Issues

**Issue:** `ModuleNotFoundError: No module named 'pkg_resources'`  
**Solution:** Upgrade setuptools: `pip install --upgrade "setuptools<70"`

**Issue:** `MatrixRankWarning: Matrix is exactly singular`  
**Solution:** This is expected for disconnected mesh fragments. The solver automatically applies Tikhonov regularization.

**Issue:** `snakemake.exceptions.WildcardError`  
**Solution:** Ensure `dataset` wildcard is specified: `snakemake --config dataset="Geometria1"`

### 9.2 Performance Tuning

```bash
# Increase parallelism
snakemake --cores 16

# Use cluster submission (SLURM)
snakemake --cluster "sbatch -n {threads}" --jobs 100

# Enable caching (requires Snakemake ≥7.0)
snakemake --use-conda --conda-prefix /path/to/conda/envs
```
